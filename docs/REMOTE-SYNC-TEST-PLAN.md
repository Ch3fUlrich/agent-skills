# Remote send + merge — test plan (local → production central)

Plan for pushing this workstation's local Omnigraph `memory` graph to the
**production** central server and reconciling both sides, **without duplicates**
and **without losing local data**. Nothing here runs automatically — it is the
checklist for the dedicated sync session.

Related: [`infra/mcp-servers/omnigraph-setup/omnigraph-sync.sh`](../infra/mcp-servers/omnigraph-setup/omnigraph-sync.sh)
(Linux), [`sync-windows.ps1`](../infra/mcp-servers/omnigraph-setup/sync-windows.ps1)
(Docker Desktop), [`omnigraph_jsonl.py`](../infra/mcp-servers/omnigraph-setup/omnigraph_jsonl.py)
(dedup/verify), and [`omnigraph-setup/README.md`](../infra/mcp-servers/omnigraph-setup/README.md).

## Decisions locked in
- **Target = production.** Writes go to central `main` (shared with other agents;
  already holds `agent-skills` + `homelab-server`). We **add** `basic-analysis` +
  `invest`. **Never** `overwrite` central — merge / branch-merge only.
- **Canary first**, then the real push.
- **No node/edge duplicates** after the merge (verified, not assumed).
- **Back up local first** — local is the newest/authoritative copy.

## Prerequisites (already staged)
- `infra/mcp-servers/omnigraph-setup/.env` (gitignored) — `CENTRAL_URL`, `CENTRAL_TOKEN`,
  `LOCAL_TOKEN`, `GRAPH=memory`, `DEVICE`. (`LOCAL_URL` there is the host value;
  the Windows helper overrides it to `http://omnigraph-server:8080` for the CLI
  container.)
- Local stack up; local graph populated (2 projects, 16 decisions, embeddings).
- Central reachable (`/healthz` 200, verified) — but **auth not yet tested**.

## Windows adaptation (why a separate helper)
`omnigraph-sync.sh` runs the CLI via `docker run --network host`, which cannot
reach `127.0.0.1:8080` on Docker Desktop. `sync-windows.ps1` instead runs the CLI
on `--network mcp-server_mcp-net`, addresses the local server as
`http://omnigraph-server:8080`, reaches central at its public URL, and pipes data
over **stdin** (no bind-mount path mangling). Both helpers share `omnigraph_jsonl.py`.

## The no-duplicates guarantee (how)
- **Nodes** dedupe by `@key` (slug) on `merge` — safe.
- **Edges** are NOT slug-keyed, so a naive cross-store `export → load` duplicates
  them. Mitigations, all built into the helpers:
  1. central side uses the **native `branch merge`** (edge-de-duplicating);
  2. after merge, `omnigraph_jsonl.py verify` **fails** the run if central `main`
     has any duplicate node (`type`+`slug`) or edge (`type`+`from`+`to`);
  3. the local pull is an **`overwrite`** of a **deduped** central export, so local
     becomes an exact, clean copy (never accumulates edges).
- Note: the local pull must be `overwrite`, not `merge` — merging nodes that carry
  vector embeddings trips a Lance ingest bug on v0.8.1; overwrite is clean. This is
  lossless because we push local → central first (central `main` ⊇ local).

## Backup / rollback
- The helpers export local `main` to `omnigraph-setup/backups/local-main-<ts>.jsonl` **before**
  any write, and restore from it if the overwrite-pull fails.
- Central is versioned: keep the `device/<host>` branch (`-KeepDeviceBranch` /
  `KEEP_DEVICE_BRANCH=1`) if you want the pre-merge state inspectable/revertible.
- Local is also re-buildable from `cluster/seed/*.jsonl` + `scripts/populate-embeddings.py`.

## Embeddings — the reconciliation (important)
- Local `Decision` vectors are **`nomic-embed-text` (768-dim)**. The provider named
  `default` in `cluster/cluster.yaml` is `nomic-embed-text`; it is **CPU-capable**
  (137M `nomic-bert`; the omnigraph README benchmarks it CPU-only ~360 ms cold /
  ~60 ms warm), so a CPU-only central can run it.
- Central's **existing** `Decision` vectors were generated by a **different**
  embedder → after merge, central `main` holds **mixed embedding spaces**, so
  `nearest()` ranking across projects would be inconsistent.
- **Reconciliation (do after the graph merge is verified):** standardize on
  `nomic-embed-text` end-to-end.
  1. Confirm/point central's query-time embedding provider at `nomic-embed-text`
     (a `coding.vm` / `Server`-repo cluster-config item — CPU-only is fine).
  2. Re-embed **all** `Decision` nodes with `nomic-embed-text` so stored vectors
     match the query-time model. Use the overwrite-on-a-branch pattern (embed →
     `load overwrite` onto a fresh branch → `branch merge`), **never** `overwrite`
     central `main` directly. `scripts/populate-embeddings.py` is the embed step;
     it will need a `--server <central>` / branch option added for central use.
  3. Verify with `search_decisions` on central.

## Step-by-step (run in the dedicated session)
1. **Auth check** (read-only): `curl -H "Authorization: Bearer $CENTRAL_TOKEN"
   https://omnigraph.ohje.ooguy.com/graphs` → expect the `memory` graph. (Tasks #2.)
2. **Snapshot BEFORE** (no writes): `sync-windows.ps1 -DryRun` (or `DRY_RUN=1
   omnigraph-sync.sh`) — backs up local, prints verify for both sides, records
   `commits_list` head on central.
3. **Canary** (via omnigraph MCP or CLI): create `Project{slug:"zzz-sync-canary"}`
   locally → run the sync → confirm it appears on central `main` and round-trips
   back to local → delete it both sides → re-verify no residue.
4. **Real push**: run the sync (no flags). It backs up local, pushes
   `device/<host>`, native-merges into central `main`, verifies central has no
   duplicates, then overwrite-pulls the deduped central back to local.
5. **Verify**: central `list_projects` shows all 4 projects with expected counts;
   `commits_list` head advanced; local + central verify **clean** (no dups).
6. **Embeddings**: reconcile per the section above; `search_decisions` on central.
7. **Cleanup**: device branch deleted by default (keep with `-KeepDeviceBranch`).

## Risks & mitigations
| Risk | Mitigation |
|---|---|
| Edge duplication | native `branch merge` + `verify` gate + overwrite-pull (see above) |
| Overwriting central | helpers only `merge`/`branch merge` central; `overwrite` is local-only |
| Overwrite-pull hits Lance index bug | restore local from backup; retry after `restart omnigraph-server` |
| Losing newest local data | backup before any write; central ⊇ local before the pull |
| Embedding-space mismatch | standardize on `nomic-embed-text`, re-embed all decisions |
| 504 from central | not a failure — verify via `commits_list` head before/after |
| Concurrent writers | device-branch merge = last-writer-wins per slug (low risk for our push) |
