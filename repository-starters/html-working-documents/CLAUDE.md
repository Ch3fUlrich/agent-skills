# Claude Project Instructions

This repository uses browser-readable HTML working documents for substantial planning, exploration, review, research, reporting, prototyping, and implementation handoff work.

## HTML Working Documents

- Use a standalone `.html` artifact when the task benefits from layout, comparison, diagrams, mockups, tables, small interactions, or durable handoff.
- Prefer markdown for short linear notes and normal repository docs that should stay easy to diff.
- Default browser artifacts to `webpage/`.
- Keep HTML self-contained: inline CSS, tiny inline JavaScript only when useful, no build step, and no remote dependencies unless explicitly justified.
- End substantial artifacts with a recommendation, implementation slice, risk table, open questions, validation plan, or export path.
- For full workflow details, read `./.codex/skills/html-working-documents/SKILL.md`.

If this is a new repository, create the first project plan as `webpage/implementation-plan.html` or another descriptive HTML file.
